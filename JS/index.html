  • undefined

Symbol([value]): 创建唯一值

console.log(Symbol('A') == Symbol('A'))   // false

BigInt: 大数据值。JS存在最大和最小的安全数字

console.log(Number.MAX_SAFE_INTEGER)  // 9007199254740991
console.log(Number.MIN_SAFE_INTEGER)  // -9007199254740991
BigInt(10)  // 10n

# 数字类型中比较奇怪的值

1、NaN 不是有效的数字，但是属于number类型

console.log(typeof NaN)  // number
console.log(NaN === NaN)  // false
Object.is(NaN === NaN)  // true

2、想检测一个值是否为有效数字 isNaN

console.log(isNaN(10))   // false
// 在检测的时候，如果当前这个值不是数字类型，先隐式转换为数字类型（Number）然后再检测是否为非有效数字  
console.log(isNaN('AA'))  // true

3、Infinity无穷大的值

console.log(Infinity)  // Infinity

parseFloat('left: 200px')  // NaN

# 检测数据类型的方法

  • typeof

typeof 检测出来的结果是字符串，字符串中包含了我们对应的数据类型; typeof 检测null的结果是'object' 不属于对象，而是因为二进制存储值以000开头了; 检测对象细分的类型，结果都是'object'

  • instanceof
  • constructor
  • Object.prototype.toString.call(value)

# 把其他数据类型转换为Number类型

  1. 特定需要转换为Number的
  • Number([val])
  • parseInt/parseFloat([val])
  1. 隐式转换（浏览器内部默认先转换为Number再进行计算的）
  • isNaN([val])
  • 数学运算

特殊情况：+在出现字符串的情况下，大部分情况是字符串拼接，+一边出现字符串或者对象，也不一定是字符串拼接，++i/i++/+i 这种情况是数学运算

let n = "10";
console.log(++n)  // 11
console.log(+n)  // 10

{} + 0  // 0
// 左边的{}认为是一个代码块，不参与运算
// 运算至处理的是 +0 => 0

0 + {}  // 参与到数学运算中 "0[object Object]"
  • 在==比较的时候，有些值需要转换为数字再进行比较
  • ...
console.log(Number(''))  // 0
console.log(Number('10'))  // 10
console.log(Number('10px'))  // NaN 只要出现数字字符结果都是NaN
console.log(Number(true))  // 1
console.log(Number(false))  // 0
console.log(Number(null))  // 0
console.log(Number(undefined)) // NaN 
console.log(Number(Symbol(10)))  // 报错
console.log(Number(BigInt(10)))  // 10

对象变为数字，应该先valueOf，没有原始值再toString变为字符串，最后把字符串转换为数字

parseInt([value])机制：从字符串左侧第一个字符开始，查找有效数字字符（遇到非有效数字字符停止查找，不论后面是否还有数字字符，都不再找了），把找到的有效数字字符转换为数字，如果一个没找到结果就是NaN（parseFloat比它多识别一个小数点）

parseInt([value],[n])把[value]看作[n]进制的数据，最后转换为十进制。

  1. [n]不写。默认是10，特殊情况字符串是以0x开头，默认值是16进制
  2. [n]范围 2～36之间，不在这个区间的，除了0和10一样，剩下结果都是NaN
parseInt("")  // NaN
Number("")  // 0
isNaN("")  // 0是有效数字 false
parseInt(null)  // 转换为字符串 parseInt('null) NaN
Number(null)  // 0
isNaN(null)  // false
parseInt('12px')  // 12
Number('12px')  // NaN
isNaN('12px')  // true
parseFloat('1.6px') + parseInt('1.2px') + typeof parseInt(null)
// 1.6 + 1 + "number"
// 2.6 + "number" => "2.6number"  在Js中加号左右两边出现字符串，则变为字符换拼接（有特殊性），如果出现对象也会变成字符串拼接（因为原本应该是把对象转换为数字，但是对象转数字需要先转换为字符串，则+遇到字符串直接变为字符串拼接）
1+[]  // 1 + "" => 1
isNaN(Number(!!Number(parseInt('0.8'))))
// parseInt('0.8') => 0
// !!0 => false
// Number(false) => 0
// isNaN(0)  => false
typeof !parseInt(null) + !isNaN(null)
// parseInt(null) => NaN
// !NaN => true
// typeof true => "boolean"
// isNaN(null) => false
// !false => true
// "booleantrue"

let arr = [10.18,0,10,25,23]
arr = arr.map(parseInt);
console.log(arr)  // [10,NaN,2,2,11]

parseInt(10.18,0)
// parseInt('10.18',0) => 10
parseInt(0,1) // NaN
parseInt(10,2)
// parseInt('10',2)
// 2进制是0和1 0和1都符合，所以是把'10'，转换为10进制
// 0*2^0 + 1*2^1 = 0 + 2 = 2
parseInt(25,3)
// parseInt('25',3)
// 3进制是0，1，2 看作3进制只有 '2'
// 2*3^0 = 2
parseInt(23,4)
// parseInt('23',4)
// 4进制 0，1，2，3 看作4进制是 '23'
// 3*4^0 + 2*4^1 = 3 + 8 = 11

# 把其他数据类型转换为字符串

  1. 能使用的办法
  • toString()
  • String()
  1. 隐式转换（一般都是调用其toString）
  • 加号运算的时候，如果某一边出现字符串，则是字符串拼接
  • 把对象转换为数字，需要先toString()转换为字符串，再去转换为数字
  • 基于alert/confirm.prompt/document.write...这些方式输出内容，都是把内容先转换为字符串，然后再输出的
  • ...

把其他类型转换为字符串，一般都是直接“”包起来，只有{}普通对象调取toString()是调取的Object.prototype.toString，不是转换为字符串，而是检测数据类型，返回结果是"[object Object]"

# 把其他数据类型转换为布尔

  1. 基于以下方式可以把其他类型转换为布尔
  • ! 转换为布尔值后取反
  • !! 转换为布尔类型
  • Boolean([val])
  1. 隐式转换
  • 在循环或者条件判断中，条件处理的结果就是布尔类型值
  • ...

规则：只有'0, NaN, null, undefined, 空字符串'五个值会变成布尔的false，其余都是true

# 在==比较的过程中，数据转换的规则：

  1. 类型一样的几个特殊点
  • {} == {} // false 对象比较的是堆内存地址
  • [] == [] // false
  • NaN == NaN // false
  1. 类型不一样的转换规则
  • null == undefined // true 但是换成===结果是false（因为类型不一样），剩下null/undefined和其他任何类型值都不相等
  • 字符串 == 对象 要把对象转换为字符串
  • 剩下如果==两边数据类型不一致，都是需要转换为数字再进行比较
// 对象 == 布尔  都转换为数字（隐式转换）
// 对象转换为数字：先toString转换为字符串（应该是先基于valueOf获得原始值，没有原始值再去toString()）,再转换为数字
console.log([] == false)  // true
// [] -> '' -> 0
// false -> 0

console.log(![] == false)  // true
// ![]  把数组转换为布尔类型然后取反 false

10 + false + undefined + [] + "js基础" + null + true + {}
// 10 + false => 10
// 10 + undefined => NaN
// NaN + [] => "NaN"
// "NaN" + "js基础" => "NaNjs基础"
// ...都是字符串拼接
// "NaNjs基础nulltrue[object Object]"

100 + 1 +21.2 + null + undefined + 'JS基础' + [] + null + 9 + false;  // NaNJs基础null9false

# 计算注意点

如果a是一个对象，与数字比较时先转换为字符串，再转换为数字
a.toString() => Object.prototype.toString() 检测数据类型

所以可以重新toString方法

var a = {
    i: 0,
    toString(){
        return ++this.i
    }
}

// 方法二：
var i = 0;
Object.defineProperty(window,'a',{
    get(){
        return ++i
    }
})
